# 1. 서비스 개요

## 1.1 서비스 목표
사용자가 보유한 종목을 기반으로,
1. **현재 포트폴리오 상태를 한눈에 파악**하고
2. **성과를 벤치마크 비교와 함께 해석**하며
3. **섹터 편중, 밸류에이션 위치, 이벤트 리스크**를 빠르게 확인하고
4. **가격/기술지표/수급/공시/뉴스 변화에 대한 알람**을 적시에 받도록 하는 것이 목표다.

# 2. 1차 서비스 상세 정의

## 2.1 포트폴리오 대시보드

### 2.1.1 보유 현황
사용자가 보유한 종목별 기본 현황을 보여준다.

#### 제공 항목
- 종목명 / 종목분야
- 보유 수량
- 평균 매입 단가
- 현재가
- 평가손익
- 수익률
- 평가금액
- 비중(포트폴리오 내 해당 종목 비중)

#### 화면 목적
- 사용자가 어떤 종목에서 수익/손실이 발생 중인지 즉시 파악
- 포트폴리오 내 특정 종목 쏠림 여부 확인 (분포도 즉시 파악)

#### 계산 로직
- 평가금액 = 현재가 × 보유수량
- 평가손익 = (현재가 - 평균매입단가) × 보유수량
- 수익률 = (현재가 - 평균매입단가) / 평균매입단가
- 비중 = 종목 평가금액 / 전체 평가금액

#### 구현 시 유의사항
- 평균 매입 단가는 사용자 입력 원장 또는 거래내역 기반으로 계산되어야 함
- 현재가는 실시간 시세 또는 장 종료 후 종가 기준으로 구분 필요
- 미국 주식이 포함되면 환율 기준 시점 정의가 필요하나, 1차에서는 국내 주식 중심으로 제한하는 것이 현실적임

---

### 2.1.2 성과 요약
전체 포트폴리오 수익률을 주요 벤치마크와 비교한다.

#### 제공 항목
- 포트폴리오 총 평가금액
- 누적 수익률
- 일간 수익률
- KOSPI / S&P 500 대비 초과수익
- 비교 차트(동일 시작일 기준 수익률 곡선)

#### 화면 목적
- “내 포트폴리오가 시장보다 잘하고 있는가?”를 직관적으로 전달
- 단순 손익 숫자보다 상대 성과 중심으로 해석 가능하게 함

#### 구현 시 유의사항
- 비교 시작일을 명확히 정의해야 함
    - 예: 사용자의 포트폴리오 생성일
    - 예: 최근 1개월 / 3개월 / YTD
- 입출금 및 매수/매도 이벤트가 있으면 단순 수익률이 왜곡될 수 있으므로
    - 1차: 단순 평가손익 기준
    - 향후: MWR, TWR 같은 정교한 수익률 방식 고려
- S&P 500 비교 시 원화 환산 여부 정책 필요
    - 1차에서는 “지수 자체 수익률 비교” 또는 “원화 환산 미적용”으로 단순화 가능

---

### 2.1.3 섹터 비중
보유 종목을 섹터별로 묶어 포트폴리오 비중을 시각화한다.

#### 제공 항목
- 섹터별 평가금액
- 섹터별 비중
- 섹터별 종목 수
- 섹터 비중 차트(도넛/막대)

#### 화면 목적
- 특정 업종 편중 여부 확인
- 리스크 분산 상태를 직관적으로 파악

#### 구현 시 유의사항
- 섹터 분류 기준 소스 필요
    - KRX / 자체 매핑 / 외부 데이터 벤더 기준 중 하나 선택
- ETF, 리츠, 스팩 등은 일반 섹터 분류와 다를 수 있어 세분화 필요

---

### 2.1.4 종목별 밸류에이션 위치
보유 종목의 현재 PER, PBR이 같은 섹터 안에서 어느 위치에 있는지 보여준다.
** PER (주가수익비율)
주가가 이익의 몇 배인지. 낮을수록 저평가, 높을수록 고평가로 보는 지표.
예) PER 10 → 지금 이익 기준으로 10년치 벌어야 주가만큼
** PBR (주가순자산비율)
주가가 순자산의 몇 배인지. 1 미만이면 청산해도 주가보다 남는다는 의미.
예) PBR 0.5 → 회사 자산을 다 팔면 주가의 2배를 받을 수 있음

#### 제공 항목
- 종목별 현재 PER
- 종목별 현재 PBR
- 동일 섹터 내 PER 분위수
- 동일 섹터 내 PBR 분위수
- 시각화 예시
    - “이 종목의 PER는 섹터 내 상위 80 분위”
    - “PBR은 섹터 평균 대비 낮은 편”

#### 화면 목적
- 현재 보유 종목이 섹터 내에서 상대적으로 고평가/저평가인지 빠르게 파악
- 절대 수치(PER 15배)보다 상대 위치를 이해하도록 지원

#### 구현 시 유의사항
- 적자 기업의 PER는 음수 또는 무의미할 수 있으므로 제외 규칙 필요
- PBR 역시 금융업/지주사/적자기업 등 해석이 다를 수 있음
- 분위수 계산의 기준 universe를 명확히 해야 함
    - 전체 상장사 기준이 아니라 **동일 섹터 유효 종목군** 기준
- 지표 갱신 주기 필요
    - 실시간 불필요, 일 배치 또는 분기 재무 업데이트 기준이면 충분
